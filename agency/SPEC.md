# herdr agency â€” format spec

An **agency** is a directory (default `.herdr/agency/`) containing a manifest,
agent profiles, and skills. The `herdr-agency` tool compiles these into a roster
and drives herdr to run them.

```
.herdr/agency/
  agency.toml                       # manifest
  agents/
    manager.md                      # orchestrator profile
    backend.md                      # worker profiles
    frontend.md
    researcher.md
  skills/
    agency-orchestrator/SKILL.md    # how the manager drives herdr
  roster.json                       # generated by `up`/`roster` (do not edit)
  inbox.jsonl                       # generated; submitted requests (one JSON/line)
  runtime.json                      # generated by `up`; live pane ids
  dispatch_state.json               # generated; dispatcher cursor (last_index)
```

## Agent profile (`agents/<name>.md`)

YAML-subset frontmatter (the same style as standard skills) plus a markdown body
that is the agent's system prompt.

| key          | type        | meaning                                                        |
|--------------|-------------|----------------------------------------------------------------|
| `name`       | string      | unique agent id (defaults to filename)                         |
| `role`       | string      | one-line description used for routing and display              |
| `model`      | string      | model label (informational; pass the real flag via `args`)     |
| `complexity` | enum        | `low` \| `medium` \| `high` â€” routing + cost tier              |
| `command`    | string      | agent CLI to launch; defaults to `agency.default_command`      |
| `args`       | list        | extra argv appended after `command` (e.g. `["--model","opus"]`)|
| `skills`     | list        | standard skills this agent uses                                |
| `tags`       | list        | keywords the orchestrator routes on                            |

Body (after the second `---`) is the agent's system prompt.

Supported frontmatter value forms: scalars (`key: value`), quoted strings, and
inline lists (`key: [a, b, c]`). Comments (`#`) and blank lines are ignored.

## Manifest (`agency.toml`)

```toml
[agency]
name = "demo-agency"
orchestrator = "manager"      # which agents/<name>.md is the manager
default_command = "claude"    # default agent CLI
default_model = "sonnet"      # default model label
workspace = "agency"          # herdr workspace label

[routing]
strategy = "manager"          # "manager" = the orchestrator LLM routes
```

## Roster (`roster.json`, generated)

The compiled, machine-readable view the orchestrator reads at runtime. One entry
per agent with `name`, `role`, `model`, `complexity`, `command`, `argv`,
`skills`, `tags`, and `orchestrator` (bool).

## Runtime model

- `herdr-agency up` creates the agency workspace and launches the orchestrator
  agent in its root pane, primed with the roster and the orchestrator skill.
- The orchestrator spins workers into sibling panes per task using
  `herdr agent start` / `herdr pane split` + `herdr pane run`, waits with
  `herdr wait agent-status`, and reads results with `herdr agent read`.
- `herdr-agency submit "<request>"` appends to `inbox.jsonl` and dispatches.

## Automatic distribution

Requests reach the agency from three places â€” `submit`, the MCP `submit_task`
tool, or a direct append to `inbox.jsonl`. The **dispatcher** forwards each new
request to the orchestrator pane exactly once:

- `pending_requests` = inbox entries past the cursor in `dispatch_state.json`.
- Delivering a request nudges the orchestrator pane and advances the cursor.
- The cursor only advances for requests actually delivered, so requests
  submitted while the agency is down stay pending until `watch`/`up --watch`
  delivers them. This makes dispatch idempotent across `submit` and `watch`.

## MCP server (`herdr-agency mcp`)

A JSON-RPC 2.0 stdio server (protocol `2024-11-05`) that lets external agentic
frameworks discover and forward to the agency. Methods: `initialize`,
`tools/list`, `tools/call`, `ping`. Tools:

| tool            | arguments              | effect                                  |
|-----------------|------------------------|-----------------------------------------|
| `submit_task`   | `{ request: string }`  | enqueue + dispatch; returns task id      |
| `agency_roster` | `{}`                   | returns the compiled roster JSON         |
| `agency_status` | `{}`                   | returns runtime, pending count, agents   |

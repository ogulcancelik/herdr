---
title: 使用 Herdr 的工作方式
description: 在本地、SSH 内或通过远程连接运行 Herdr。
---

在哪里工作，就在哪里运行 Herdr。人在哪里，就从哪里连接。

Herdr 是一个后台 Session 服务器，外加一个或多个终端客户端。Pane 在服务器里持续运行。客户端负责接入、断开和渲染 Session。

## 本地工作

在项目目录中启动 Herdr：

```bash
herdr
```

Herdr 会自动启动或接入本地后台 Session。你不用管 socket。在 Pane 里照常跑 shell、服务器、测试和 Agent。

按 `ctrl+b q` 断开客户端，Pane 继续运行。

之后重新接入：

```bash
herdr
```

如果想结束 Session 并停止其中的 Pane，停掉服务器：

```bash
herdr server stop
```

## 通过普通 SSH 的远程工作

SSH 到有代码和凭据的机器，然后在那里运行 Herdr：

```bash
ssh you@server
herdr
```

这和终端复用器的用法一样。shell 在远端，Herdr 服务器在远端，Agent 和 Pane 都跑在远端机器上。按 `ctrl+b q` 断开连接，之后再 SSH 回去，重新跑 `herdr`。

如果你本来就在 SSH shell 里工作，或者在用手机、平板的 SSH 客户端，或者想要最简单的配置，走这条路。

## 用手机工作

你不需要 Herdr 手机 App 或 Web 仪表盘。手机上装个 SSH 客户端，连到 Agent 所在的机器，在那里启动 Herdr：

```bash
ssh you@server
herdr
```

同一个持久的 Herdr Session 就在手机终端里打开。TUI 会适配窄屏，你不用离开 SSH 就能查看 Agent、切换 Workspace、看 Pane。

## 从本地终端远程工作

不用先开 shell，直接通过 SSH 连接：

```bash
herdr --remote workbox
herdr --remote ssh://you@server:2222
```

你本地的 Herdr 充当瘦客户端。它通过 SSH 连接，启动或接入远程 Herdr 服务器，把 UI 流回本地终端。

想让远程 Session 用起来像本地的，走这条路。客户端跑在你自己的机器上，所以图像剪贴板粘贴这类本地桌面功能能桥接到远程服务器。如果先 SSH 再在服务器上跑 `herdr`，Herdr 就完全跑在那台服务器上，读不到你本地桌面的剪贴板。

经常连的目标，把主机写进 SSH 配置：

```text
Host workbox
  HostName server.example.com
  User you
  Port 2222
```

然后这样连：

```bash
herdr --remote workbox
```

## 该用哪条路径

本地工作用 `herdr`。想让 Herdr 在远程 shell 上像 tmux 一样工作，或者在用手机 SSH 客户端时，先 `ssh you@server` 再 `herdr`。想要一个连远程 Session 的本地瘦客户端（包括本地剪贴板图像粘贴桥接），用 `herdr --remote <host>`。

远程引导细节、命名远程 Session、自定义二进制、直接终端接入和 `--no-session`，参见[持久化与远程访问](/zh-cn/docs/persistence-remote/)。

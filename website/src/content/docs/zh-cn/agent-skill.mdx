Herdr 提供了一个可复用的 Agent 技能文件：[`SKILL.md`](https://github.com/ogulcancelik/herdr/blob/master/SKILL.md)。

把这个文件安装到任何支持可复用技能或自定义指令的编程 Agent 中。这份技能教 Agent 如何在 Herdr Pane 内操作 Herdr。

Herdr 还提供了另一份用途不同的指南：[`herdr.dev/agent-guide.md`](https://herdr.dev/agent-guide.md)，用于让 Agent 帮助人类学习、安装或排查 Herdr。技能是给操作 Herdr 的 Agent 用的；指南是给教人类的 Agent 用的。

## 这份技能做什么

这份技能告诉 Agent：当 `HERDR_ENV=1` 已设置时，使用 `herdr` CLI。这意味着 Agent 运行在 Herdr 管理的 Pane 内，可以安全地与本地 Herdr socket 通信。

安装技能后，Agent 可以：

- 查看 Workspace、Tab、Pane 和相邻的 Agent
- 拆分 Pane，运行命令而不抢占焦点
- 读取 Pane 输出和最近的日志
- 等待服务器、测试或其他 Agent 完成
- 在相邻 Pane 中启动辅助 Agent

这份技能不是独立的应用或服务，它只是一份给 Agent 用的 Markdown 指令文件。

## 安装

用 `npx skills` 安装技能：

```bash
npx skills add ogulcancelik/herdr --skill herdr -g
```

`-g` 参数为受支持的 Agent 全局安装。省略 `-g` 则安装到当前项目。

仓库中的副本可作为手动兜底方案和事实来源：

```text
https://github.com/ogulcancelik/herdr/blob/master/SKILL.md
```

如果 Agent 支持技能系统，把这个文件安装为名为 `herdr` 的技能。如果不支持，把文件内容粘贴到该 Agent 的项目指令或用户指令中。

安装完成后，在 Herdr 内启动 Agent：

```bash
herdr
claude
```

也可以在 Herdr Pane 里使用其他任何编程 Agent。关键是 Agent 进程要运行在 Herdr 内部，这样 `HERDR_ENV=1` 才可用。

## 安全规则

这份技能开头设有一条安全护栏：如果 `HERDR_ENV=1` 未设置，Agent 应当停止操作，并说明自己没有运行在 Herdr 管理的 Pane 内。

这样可以避免 Herdr 外部的 Agent 试图控制不属于它的 Session。

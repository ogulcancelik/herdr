Herdr 原生支持鼠标。你可以点击 Pane、Tab、Workspace 和 Agent，拖动分割边框，使用右键菜单，一个按键绑定都不用学。键盘操作是可选层，不是必需项。

## 前缀键是什么

终端复用器位于终端和其中运行的程序之间。这些程序已经占用了大多数按键组合：`ctrl+c` 中断，`ctrl+r` 搜索历史，编辑器几乎占满了剩下的。如果 Herdr 直接抢用常用按键，就会干扰里面的程序。

前缀键解决了这个问题。按下前缀键（默认 `ctrl+b`），下一次按键发给 Herdr 而非终端。`prefix+c` 的意思是：按下 `ctrl+b`，松开，再按 `c`。只保留一个专用键，而不是几十个。

随时按 `prefix+?` 可以查看所有生效的绑定。

## 先学这五个

| 动作 | 按键 |
| --- | --- |
| 新建 Tab | `prefix+c` |
| 向右 / 向下分割 | `prefix+v` / `prefix+minus` |
| 在 Pane 间移动 | `prefix+h/j/k/l` |
| Workspace 导航 | `prefix+w` |
| 分离并让一切继续运行 | `prefix+q` |

这五个覆盖了日常大部分操作。其余的用鼠标就行。

## 其余按键，按任务分

Pane：

| 动作 | 按键 |
| --- | --- |
| 缩放聚焦 Pane | `prefix+z` |
| 关闭 Pane | `prefix+x` |
| 交换 Pane | `prefix+shift+h/j/k/l` |
| 调整大小模式 | `prefix+r` |
| 复制模式 | `prefix+[` |

Tab：

| 动作 | 按键 |
| --- | --- |
| 下一个 / 上一个 Tab | `prefix+n` / `prefix+p` |
| 跳转到 Tab 1–9 | `prefix+1..9` |
| 重命名 Tab | `prefix+shift+t` |
| 关闭 Tab | `prefix+shift+x` |

Workspace 与 Session：

| 动作 | 按键 |
| --- | --- |
| 新建 Workspace | `prefix+shift+n` |
| 重命名 Workspace | `prefix+shift+w` |
| 关闭 Workspace | `prefix+shift+d` |
| Goto 选择器 | `prefix+g` |
| 切换侧边栏 | `prefix+b` |

完整键位和绑定语法见[按键绑定参考](/zh-cn/docs/configuration/#按键绑定)。

## 复制模式

按 `prefix+[` 让聚焦的 Pane 进入复制模式。移动可用 `h/j/k/l`、`w/b/e`、`{`/`}`、`PageUp`/`PageDown`、`ctrl+b`/`ctrl+f`、`ctrl+u`/`ctrl+d`。开始选用 `v` 或空格，复制用 `y` 或回车，不复制直接退出用 `q` 或 Esc。

你设置的前缀键在复制模式里仍然生效。用默认前缀时，`ctrl+b` 会进入前缀模式而不是向上翻页。如果想用 `ctrl+b` 在复制模式里翻页，换一个前缀键。

鼠标拖选完全不用进复制模式，直接就能复制。

## 一切都可以改

每个绑定都可以配置，包括前缀键本身：

```toml
[keys]
prefix = "ctrl+a"
```

## 不用前缀键

你可以把 Herdr 操作绑定到不需要前缀的直接组合键。难的是搞清楚哪些组合键安全——终端、shell 和桌面环境已经占了键盘大半。

任何组合键都能做绑定：`ctrl+j`、`alt+k`，顺手就行。但一个组合键要穿过三层才能到达 Herdr：操作系统、外层终端（Ghostty、iTerm2 等都自带默认键位），还有 Pane 里运行的程序。`ctrl+j` 能顺利到达 Herdr，但 shell 和编辑器把它当作回车。`alt+k` 在 Linux 上空闲，在 macOS 大多数终端里会合成特殊字符。从这些键系里选组合键的话，最好对照自己终端和系统的快捷键再确认一遍。

我们梳理了 Ghostty、iTerm2、Terminal.app、kitty、WezTerm、Alacritty、Warp、Windows Terminal、GNOME Terminal 和 Konsole 的默认绑定，以及 GNOME 和 KDE 的全局快捷键。有一个修饰键系几乎在所有地方都空着：`ctrl+alt`。终端不碰它，macOS 的 option 键合成行为（会挡掉纯 `alt` 组合键）也影响不到它，哪怕终端不支持现代键盘协议也能正常传输。它是稳妥的默认选择，用不用随你。

下面的配置保留前缀绑定，并在此基础上叠加直接组合键：

```toml
[keys]
focus_pane_left = ["prefix+h", "ctrl+alt+h"]
focus_pane_down = ["prefix+j", "ctrl+alt+j"]
focus_pane_up = ["prefix+k", "ctrl+alt+k"]
focus_pane_right = ["prefix+l", "ctrl+alt+l"]
previous_tab = ["prefix+p", "ctrl+alt+["]
next_tab = ["prefix+n", "ctrl+alt+]"]
new_tab = ["prefix+c", "ctrl+alt+c"]
split_vertical = ["prefix+v", "ctrl+alt+d"]
split_horizontal = ["prefix+minus", "ctrl+alt+shift+d"]
zoom = ["prefix+z", "ctrl+alt+z"]
```

少数 `ctrl+alt` 组合键已被其他功能占用。请避开这些：

| 组合键 | 占用者 |
| --- | --- |
| `ctrl+alt+arrows` | GNOME Workspace 切换，Ghostty 和 Konsole 默认键位 |
| `ctrl+alt+t` | Ubuntu 和 Fedora 的"启动终端" |
| `ctrl+alt+l` / `ctrl+alt+a` | KDE 锁屏 / 注意力窗口 |
| `ctrl+alt+s` / `ctrl+alt+u` | Konsole |
| `ctrl+alt+f1..f12` | Linux 虚拟控制台切换 |

如果某个直接组合键没反应，说明终端或桌面环境在 Herdr 收到之前就把它拦截了。任选一边重新配置就行：在终端设置里释放这个组合键，或者在 Herdr 里换一个。

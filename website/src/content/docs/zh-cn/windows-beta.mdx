Herdr 对 Windows 的原生支持目前处于实验性测试阶段。

在 Windows 上，Herdr 改用 ConPTY 和 Windows 的进程与运行时机制，而非最初基于 Unix PTY 模型的设计。部分功能能自然适配 Windows，部分则不能。此预览不承诺 Linux/macOS 上的所有功能都能在 Windows 上完整支持。

测试版的目标是从真实使用中收集反馈：安装成功率、Pane 稳定性、Agent 工作流、bug 数量、功能缺口，以及 Windows 用户能否从 Herdr 获得足够价值。根据这些反馈，Windows 支持可能升级为稳定版，或在成熟前维持预览状态，也可能因维护成本过高而缩减。

使用 PowerShell 安装 Windows 原生测试版：

```powershell
powershell -ExecutionPolicy Bypass -c "irm https://herdr.dev/install.ps1 | iex"
```

Windows 测试版仅通过预览通道发布。安装器默认选用预览通道，会将 `channel = "preview"` 写入 Herdr 配置，在 `%USERPROFILE%\.herdr\packages\standalone\releases` 下保存各版本，让 `%LOCALAPPDATA%\Programs\Herdr\bin` 指向当前版本，并保留少量旧版本以防运行中的进程阻塞更新。

内部测试时，可以设置 `HERDR_MANIFEST_URL` 让安装器指向自定义清单，而非 Herdr 的稳定或预览清单。

| 能力 | 状态 |
| --- | --- |
| 本地持久 Session | 测试版 |
| 通过 ConPTY 的原生 Pane | 测试版 |
| Windows Terminal / PowerShell 应用连接 | 测试版 |
| `cmd.exe` Pane | 测试版 |
| 启动 cwd 和 Workspace 标签 | 测试版 |
| Pane 启动 cwd | 测试版 |
| Agent 命令发现 | 测试版 |
| Agent 自报告集成 | 测试版 |
| Agent 进程树检测 | 测试版 |
| 从已知 cwd 检测 Git/worktree | 测试版 |
| 插件 | 预览 |
| Pane 屏幕历史 | 测试版 |
| 嵌套启动覆盖 | 测试版 |

Windows 的 Agent 进程检测会扫描 Pane shell 的后代进程，识别直接运行的 Agent 和常见的命令包装器。它对 Codex、Claude 等 Agent 很有用，但与 Unix 前台进程组检测不是一回事。

插件目前以预览状态支持在清单中声明 `windows` 平台。GitHub 安装、本地链接、构建命令、动作、事件和插件 Pane 在 Windows 上均为尽力而为。命令为 argv 命令，必须兼容 Windows；`npm`、`bun`、`node` 等 Node 包 shim 只要在 `PATH` 中就能使用，而使用 `sh` 或 Bash 的 Unix 专用示例需要 Windows 专用替代方案。平台过滤器会跳过不支持的构建命令，并对不支持的动作或 Pane 返回 `platform_unsupported`。

## 部分支持

| 能力 | 状态 |
| --- | --- |
| shell `cd` 之后的实时 cwd | 部分支持 |
| 通过 shell 集成/OSC7 的实时 cwd | 测试版 |
| 向 Agent 粘贴剪贴板图像 | 未验证 |
| CJK 隐藏光标显示 | 测试版 |
| Kitty graphics 渲染 | 未验证 |

Herdr 能在正确目录中启动 Pane，并可从你启动 Herdr 的目录创建初始 Workspace。但启动后 PowerShell 的目录变化就不一样了：Herdr 可读取的进程字段无法可靠跟踪后续的逻辑 `cd` 变更。如需实时 cwd 上报，请使用 Herdr 集成或提示符 shell 集成。

Windows Terminal 可能为特定 Agent 提供图像粘贴路径，但 Herdr 自身的剪贴板图像读取器尚未在 Windows 上接通。在 Windows 剪贴板桥实现并测试之前，请将 `alt+v` 图像粘贴视为未验证。远程剪贴板图像桥是独立功能，仍绑定于 Unix/macOS 的 `herdr --remote`。

Kitty graphics 仍是实验性功能，尚未宣称支持 Windows。除非你专门在 Windows Terminal 中测试图像渲染，否则请保持 `experimental.kitty_graphics = false`。

## 已知注意事项

### 光标渲染

Windows 终端通过 ConPTY 运行 Herdr。在全屏重绘期间，原生终端光标可能闪烁、跳动或短暂显示旧位置。Herdr 默认的 `host_cursor = "auto"` 会在 Windows 上将 Herdr 光标绘制为终端单元格内容，而 Linux 和 macOS 继续使用原生终端光标。Windows 上的取舍是：在 Herdr 内获得稳定不闪烁的光标，但无法使用外层终端的原生闪烁、形状和光标颜色。

如需在 Windows 上改回外层终端光标，请设置：

```toml
[ui]
host_cursor = "native"
```

### 键盘和鼠标

Windows 终端对带修饰键的按键报告方式不尽相同。Herdr 在 Windows Terminal 和 Windows 版 Alacritty 中保留鼠标报告和 `ctrl+j`，但 `shift+enter` 只有在外层终端将其报告为独立的修饰 Enter 键时才有效。如果 Windows 或终端将其报告为普通 Enter，Herdr 就转发普通 Enter。

## 复制与粘贴

Herdr 的 Pane 文本复制在 Windows 测试版上可用。在 Pane 内拖选文本即可通过 Herdr 复制。

文本粘贴请在 Windows Terminal 中使用 `ctrl+shift+v`。多行文本粘贴采用带括号模式（bracketed），因此 shell 和 Agent 提示符会将其作为一次粘贴接收，而非逐行提交。按住 `shift` 再右键，可以使用外层终端的粘贴操作，而不将点击发给 Herdr。

## Windows 测试版不支持

| 能力 | 状态 |
| --- | --- |
| 直接终端附加 | 不支持 |
| Windows 二进制的 `herdr --remote` | 不支持 |
| 实时服务器交接 | 不支持 |
| Unix 文件描述符交接 | 不支持 |
| Unix 前台进程组 | 不支持 |
| 远程剪贴板图像桥 | 不支持 |
| 前缀输入法切换 | 不支持 |
| 签名二进制 / 规避 SmartScreen | 不支持 |

在 Windows 上远程工作时，请 SSH 到服务器并在那里运行 `herdr`：

```powershell
ssh you@server
herdr
```

此模式下 Herdr 运行在远程主机上。Windows 原生的 `herdr --remote` 不在测试版范围内。

Windows 更新通过 Windows 安装器执行，并更新带版本号的安装联接点。更新后请重启运行中的 Herdr Session。实时交接仅限 Unix。

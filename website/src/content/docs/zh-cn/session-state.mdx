Herdr 有好几条状态路径，各自解决不同的问题。

## 什么能保留下来

| 场景 | 进程继续运行 | 布局恢复 | 最近的屏幕内容恢复 | Agent 对话恢复 |
| --- | --- | --- | --- | --- |
| 分离并重新连接 | 是 | 是 | 是，来自实时终端 | 是，因为进程从未停止 |
| 服务器重启 | 否 | 是 | 仅在开启 Pane 屏幕历史时 | 仅在有 Agent 原生 Session 恢复时 |
| 不带 `--handoff` 的更新 | 兼容的服务器继续运行；需要重启的服务器可能要停止/重启 | 重启后恢复 | 仅在开启 Pane 屏幕历史时 | 仅在有 Agent 原生 Session 恢复时 |
| 带 `--handoff` 的更新 | 对受支持的运行中服务器尽力而为 | 是 | 是，交接成功时来自实时终端 | 是，交接成功时进程保持运行 |

下面逐一解释每条路径。

## 实时持久化

普通分离后，Herdr 服务器继续运行。Pane、shell、Agent、服务器、测试和命令进程都在里面跑着。

用 `ctrl+b q` 分离客户端，之后重新连接：

```bash
herdr
```

这是最强的持久化路径，因为原始进程从未停止。

## 快照恢复

如果 Herdr 服务器停了再启动，原来的 Pane 进程就没了。Herdr 会恢复保存的 Session 形态：Workspace、Tab、Pane、cwd、布局和焦点。

快照恢复不会保留运行中的 shell、服务器、测试或其他进程。无法走更强恢复路径的 Pane，会在各自保存的目录中以新 shell 的形式回来。

## Pane（窗格）屏幕历史回放

Pane 屏幕历史在服务器完全重启后，恢复最近的终端内容。它恢复的是 Herdr 能展示的东西，不是原来的进程。

默认关闭，因为 Pane 输出可能包含密钥、令牌、提示符和命令输出。在 Settings > Experiments > pane screen history 中开启，或者：

```toml
[experimental]
pane_history = true
```

开启后，Herdr 把 Pane 历史存到 `session.json` 旁边的 `session-history.json`。请像对待终端历史一样对待 Herdr 的配置/Session 目录。

## Agent（智能体）原生 Session 恢复

有些 Agent 能恢复自己的对话 Session。Herdr 可以利用官方集成上报的 Session 引用，在服务器重启后重新启动受支持的 Agent Pane。

默认开启。关闭方法：

```toml
[session]
resume_agents_on_restore = false
```

Herdr 只恢复通过当前官方集成上报了原生 Session 引用的 Pane。

客户端连接后，提供了终端尺寸和主题上下文，Herdr 就会跨 Workspace 和 Tab 恢复符合条件的 Agent Pane，不用等每个 Pane 被聚焦。

Agent 原生 Session 恢复需要以下版本或更新的 Herdr 集成：

| Agent | 最低 Herdr 集成版本 | 恢复命令 |
| --- | --- | --- |
| Pi | `2` | `pi --session <path-or-id>` |
| OMP | `3` | `omp --resume=<path-or-id>` |
| Claude Code | `6` | `claude --resume <id>` |
| Codex | `5` | `codex resume <id>` |
| Cursor Agent CLI | `1` | `cursor-agent --resume <id>` |
| GitHub Copilot CLI | `2` | `copilot --resume=<id>` |
| Devin CLI | `2` | `devin --resume <id>` |
| Droid | `2` | `droid --resume <id>` |
| Kimi Code CLI | `3` | `kimi --session <id>` |
| Qoder CLI | `2` | `qodercli --resume <id>` |
| OpenCode | `5` | `opencode --session <id>` |
| Kilo Code CLI | `1` | `kilo --session <id>` |
| Hermes Agent | `2` | `hermes --resume <id>` |
| MastraCode | `1` | `mastracode --thread <id>` |

运行 `herdr integration status` 查看已安装的集成版本。用 `herdr integration install <agent>` 重新安装过期的集成。

不受支持、缺失、无效、重复或过期的 Session 引用，会在保存的 Pane 目录中作为普通 shell 恢复。

如果某个 Pane 适用 Agent 原生 Session 恢复，Herdr 会恢复该 Pane 的 Agent Session，而不是回放保存的 Pane 历史。

## 实时交接

实时交接用于需要替换运行中 Herdr 服务器的更新和远程连接场景。它让旧服务器把实时 Pane 转移给新服务器，使 Pane 进程在服务器替换后继续运行。

这不同于快照恢复、Pane 历史回放和 Agent 原生 Session 恢复。交接尝试让当前进程继续活着，其他路径则是在旧服务器已停止后重建状态。

实时交接是实验性功能，需要主动开启：

```bash
herdr update --handoff
herdr --remote workbox --handoff
```

普通的 `herdr update` 和 `herdr --remote workbox` 默认走常规的重启/停止流程。

`herdr update --handoff` 只适用于 Herdr 自带更新器管理的安装。Homebrew、mise 和 Nix 安装通过各自的包管理器更新，所以这些安装中 `herdr update` 被禁用，无法执行实时交接。

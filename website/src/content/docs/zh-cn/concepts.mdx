---
title: 核心概念
description: 理解 Herdr 的 Workspace、Tab、Pane、Agent、Session 和模式。
---

Herdr 是一个终端 Workspace 管理器。它让真实的终端进程持续运行，并围绕它们添加结构。

## Workspace（工作区）

Workspace 是最顶层的项目容器。每个仓库、任务或调研使用一个 Workspace。

Workspace 拥有 Tab 和 Pane。它在侧边栏中的状态由内部的 Agent 汇总而来，让你一眼看出哪个项目需要关注。

## Tab（标签页）

Tab 是 Workspace 内的布局。用 Tab 来分隔不同视图，比如 `agents`、`logs`、`server` 或 `review`。

可以通过 CLI 和 Socket API 对 Tab 进行寻址。

## Pane（窗格）

Pane 是真实的终端。Herdr 渲染终端输出、把输入传回进程，客户端断开后 Pane 仍然保留。

Pane 可以向右或向下分割。你可以手动重命名 Pane、通过 CLI 读取、发送输入或关闭它。

## 鼠标 UI

Herdr 原生支持鼠标操作。你可以点击 Pane、Tab、Workspace 和 Agent，拖动分割边框、选择文本、使用右键菜单。下面介绍的所有操作都可以用鼠标完成；按键绑定只是可选的一层。

如果你偏好纯键盘操作，或者不想让 Herdr 捕获鼠标输入，可以禁用鼠标捕获：

```toml
[ui]
mouse_capture = false
```

## Agent（智能体）

Agent 是 Herdr 在 Pane 内识别出的进程。Herdr 通过前台进程、屏幕清单以及可选的集成来检测 Agent。

Agent 状态如下：

| 状态 | 含义 |
| --- | --- |
| `blocked` | Agent 需要输入、审批或决策。 |
| `working` | Agent 正在运行。 |
| `done` | Agent 已完成，你还没有查看。 |
| `idle` | Agent 已完成或在等待，且已被查看。 |
| `unknown` | Herdr 无法有把握地判断状态。 |

## Session（会话）

Session 是一个持久的 Herdr 服务器命名空间。直接运行 `herdr` 命令会连接到默认 Session。

命名 Session 是相互独立的运行时命名空间：

```bash
herdr session list
herdr session attach work
herdr session attach side-project
```

优先使用 Workspace。只有当你需要完全独立的 Pane、Socket 和持久化运行时状态时，才使用命名 Session。

## 客户端与服务器

默认情况下，Herdr 以后台服务器加一个或多个已连接客户端的方式运行。

服务器持有 Pane 和进程状态。客户端是连接到该服务器的终端 UI。

按 `ctrl+b q` 分离客户端。服务器和 Agent 会继续运行。

如果想结束 Session 并停止其中的 Pane，停止服务器：

```bash
herdr server stop
```

## 模式

Herdr 有终端模式、前缀模式和导航模式。

终端模式把按键发送给当前聚焦的 Pane。前缀模式在按下前缀键后等待一个 Herdr 动作。导航模式是常驻的 Workspace 导航界面。

按下前缀键（默认 `ctrl+b`），再按一个动作键，比如 `c` 新建 Tab，`w` 打开 Workspace 导航。不熟悉前缀键？参见[键盘](/zh-cn/docs/keyboard/)。

Herdr 可以同时运行多个编程 Agent。每个 Agent 都有自己的终端 Pane，里面的 shell、日志、提示符和运行中的进程都保持原样。Herdr 会跟踪哪些 Pane 里有 Agent，把状态汇总到 Tab 和 Workspace 层级，让你直接跳到需要关注的 Pane，不用手动逐个翻终端。

## 受支持的 Agent

常见编程 Agent 开箱即可自动检测。重要的不是 Herdr 能不能看到某个 Agent，而是哪个信号有权决定 `idle`、`working` 和 `blocked`。

## 状态权威

Herdr 先检测每个 Pane 的前台进程。每个 Pane 有且只有一个状态权威。

对于有完整生命周期钩子的 Agent，当集成已安装且正在为当前 Pane 主动上报时，集成就是权威来源。Herdr 用钩子上报来判定 `idle`、`working`、`blocked` 和 Session 身份。同一个生命周期权威不会再并行跑屏幕清单兜底，避免出现两个互相冲突的事实来源。

对于没有完整生命周期钩子的 Agent，Herdr 先识别前台进程，再读取底部缓冲区的实时屏幕快照，然后用 TOML 清单对快照做评估，判定 `idle`、`working` 和 `blocked`。对于会发出这些信号的 Agent，清单还可以匹配终端标题和进度（OSC）序列作为检测证据；没有这些证据时，单靠屏幕规则完成检测。

屏幕快照来自 Pane 缓冲区的底部最新内容，而不是滚动后的视口。即使你在 Herdr 里向上翻页，检测仍然跟随底部的实时 Agent UI。

Claude Code、Codex、GitHub Copilot CLI、Droid、Qoder CLI 和 Cursor Agent CLI 的集成有意不设为生命周期权威。它们为恢复提供原生 Session 身份，但钩子并不覆盖整个生命周期，可能漏掉权限审批结果、Esc 中断或其他状态转换。对这些 Agent，Herdr 仍然使用屏幕清单检测。

## 虚拟机与沙箱包装器

在 Linux 上，VM、Bubblewrap 或 `fence` 这类包装器可能把真实的 Agent 进程从宿主 `/proc` 中隐藏掉。在命令前加上 `HERDR_AGENT=<agent>`，例如 `HERDR_AGENT=claude fence -- claude`，告诉 Herdr 该用哪个已有 Agent 的屏幕清单。这个提示只对当前前台进程生效；除非你希望所有继承它的前台进程都被当作该 Agent，否则不要在 shell 里全局 export。

## blocked 状态

对于屏幕清单类 Agent，`blocked` 检测刻意从严。只有当实时底部缓冲区快照匹配已知的审批、提问或权限 UI 时，Herdr 才标记 `blocked`。对已知 Agent，如果没有任何清单规则匹配，Herdr 会回退到 `idle`，并在 explain 输出中标记为 `default_known_agent_idle_fallback`。

这意味着不常见的新 Agent 提示可能一开始显示为 `idle` 而非 `blocked`，要等 Herdr 学会那种屏幕形态。这些交互不会让 Herdr 发送输入或执行破坏性操作，只影响可见状态和等待。

## 检测清单

内置清单已打包在 Herdr 中。Herdr 还会向 herdr.dev 检查远程清单更新，自动应用各 Agent 的有效规则更新，无需重启 Herdr。远程清单存放在 Herdr 的状态目录中。设置 `[update] manifest_check = false` 可以禁用后台远程清单检查。

本地覆盖文件放在平台配置目录中，可以替换远程或内置清单：

```text
~/.config/herdr/agent-detection/<agent>.toml
```

本地覆盖始终优先。没有本地覆盖时，Herdr 会在缓存的远程清单和当前运行二进制内置的清单之间，选择较新且兼容的版本。调试构建中，同一个配置助手可能使用 `herdr-dev` 之类的开发目录。无效的覆盖文件会被忽略并给出警告，Herdr 会对该 Agent 回退到缓存的远程或内置清单。

远程清单只为 Herdr 已经能识别的 Agent 修补检测规则。添加全新的 Agent 仍然需要更新 Herdr 二进制文件，以获得进程检测、标签和集成行为。

运行中的服务器启动时会把生效清单加载到内存。远程清单自动更新后，会重新加载这个内存缓存。运行 `herdr server update-agent-manifests` 可以立即拉取远程清单更新，并重载运行中的服务器。手动编辑本地覆盖文件后，重启 Herdr 或运行 `herdr server reload-agent-manifests` 即可应用到运行中的服务器。

当某个 Pane 显示了错误状态，可以用 `herdr agent explain` 排查：

```bash
herdr agent explain <target>
herdr agent explain --file screen.txt --agent codex --json
```

实时 explain 由运行中的服务器评估，反映的是当前生效的清单缓存。输出内容包括：Agent 名称、最终状态、屏幕检测是否被完整生命周期权威跳过、清单来源和版本、缓存的远程版本、本地覆盖的遮蔽情况、远程更新状态、匹配的规则、可见证据标志、已评估规则的匹配器和区域证据、转写查看器的跳过更新原因，以及没有规则匹配时的 `idle` 回退原因。

Herdr 可以在 tmux 作为外层终端环境时运行。Agent 检测不会检查 Herdr Pane 内启动的 tmux 会话。如果某个 shell 框架在 Herdr 内自动进入 tmux，Herdr 看到的 Pane 进程就是 `tmux`，而不是背后的 Agent。

## 状态汇总

侧边栏会把状态向上汇总。

一个 `blocked` 的 Agent 会让它的 Pane、Tab 和 Workspace 都显示为 blocked。一个 `working` 的 Agent 会让 Workspace 显示为活跃。一个 `done` 的 Agent 在你查看之前会一直保持可见。

这就是 Herdr 的核心工作流：启动多个 Agent 并行运行，通过侧边栏一眼看出哪个项目需要决策、哪个还在跑、哪个已经可以审阅。

## 直接集成

为每个你使用的 Agent 安装集成。集成让 Herdr 获得钩子或插件上报，不再只靠屏幕检测：

```bash
herdr integration install claude
herdr integration status
```

每个受支持的 Agent 都有自己的集成名称和行为。各 Agent 的细节和完整安装列表见[集成](/zh-cn/docs/integrations/)。

## 自定义 Agent 标签

你可以重命名 Agent 目标的显示名称：

```bash
herdr agent rename w1:p1 reviewer
herdr agent rename reviewer --clear
```

目标可以是终端 ID、唯一 Agent 名称、检测到或上报的 Agent 标签，以及旧式 Pane ID。

## 自定义状态标签

集成可以上报一个视觉状态标签，不改变语义状态。

```bash
herdr pane report-agent w1:p1 \
  --source custom:indexer \
  --agent docs-bot \
  --state working \
  --custom-status indexing
```

`state` 控制等待、通知和汇总。`custom-status` 只是显示文本。

## 从 CLI 启动 Agent

当你想把一个终端当作 Agent 目标时，使用 `herdr agent ...` 命令。Agent 目标会出现在 `agent list` 中，可以按 Agent 名称读取或发送输入、按状态等待，也可以直接附加。

从脚本向 Herdr 中启动一个 Agent：

```bash
herdr agent start reviewer --cwd ~/project --split right -- pi
```

也可以把该 Agent 放进特定的 Workspace 或 Tab：

```bash
herdr agent start docs --workspace w1 --tab w1:t1 -- claude
```

普通终端、服务器、测试、shell 和底层终端输入，请使用 `herdr pane ...` 命令。例如跑 `cargo test` 用 `pane split` 和 `pane run`，不要用 `agent start`，除非那个终端就是有意要作为 Agent 目标。

## 直接附加到 Agent

把当前终端附加到某一个 Agent 终端，而非完整的 Herdr UI：

```bash
herdr agent attach reviewer
```

用 `ctrl+b q` 分离。用 `ctrl+b ctrl+b` 发送字面的 `ctrl+b`。

用鼠标滚轮或普通的 `page up`/`page down` 滚动。正常输入会自动跳回底部。

如果另一个直接附加客户端已经持有输入控制权，用 `--takeover`：

```bash
herdr agent attach reviewer --takeover
```

想对非 Agent 终端获得相同的直接附加效果时，用 `herdr terminal attach <terminal_id>`。

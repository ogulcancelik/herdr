Herdr CLI 通过本地 Socket API 与运行中的服务器通信，集成和 Agent 用的也是同一条通道。

大多数命令返回 JSON 响应。需要确定性自动化时，可以在脚本中调用。

## 启动与状态

```bash
herdr                         # 启动或连接默认 Session
herdr --session work          # 启动或连接命名 Session
herdr --remote workbox        # 通过 SSH 连接,使用本地按键绑定
herdr --remote workbox --remote-keybindings server
herdr --remote workbox --handoff
herdr --no-session            # 单进程逃生舱
herdr --default-config        # 打印默认配置
herdr update                  # 从配置的通道下载并安装
herdr update --handoff        # 对受支持的运行中服务器启用实时交接
herdr completion zsh          # 生成 zsh 补全脚本
herdr channel show            # 打印 stable 或 preview
herdr channel set preview     # 启用预览构建
herdr channel set stable      # 把 Linux/macOS 直接安装切回稳定版
herdr --version               # 打印版本
```

状态相关命令：

```bash
herdr status
herdr status server
herdr status client
```

API schema 相关命令：

```bash
herdr api schema
herdr api schema --json
herdr api schema --output herdr-api.schema.json
```

`herdr api schema` 打印随安装包附带的 socket 协议 schema 摘要。加 `--json` 输出完整 JSON Schema 文档，加 `--output PATH` 写入文件。

## Shell 补全

```bash
herdr completion zsh
herdr completions zsh
herdr completion bash
herdr completion fish
herdr completion powershell
herdr completion elvish
```

`completion` 把脚本输出到 stdout。`completions` 是别名。临时 zsh 会话直接加载脚本即可：

```bash
source <(herdr completion zsh)
```

持久化 zsh 配置时，把生成的 `_herdr` 函数写到 `compinit` 执行前已在 `fpath` 中的路径：

```bash
mkdir -p ~/.zfunc
herdr completion zsh > ~/.zfunc/_herdr
```

然后确保 `.zshrc` 中有：

```zsh
fpath=(~/.zfunc $fpath)
autoload -Uz compinit
compinit
```

## 服务器

```bash
herdr server
herdr server stop
herdr server reload-config
herdr server agent-manifests [--json]
herdr server update-agent-manifests [--json]
herdr server reload-agent-manifests
```

`herdr server` 显式启动无界面服务器，适合受管或服务化部署场景。
`reload-config` 应用可热重载的设置，不重启 Pane。
`agent-manifests` 显示当前生效的 Agent 检测清单来源、缓存的远程版本，以及最近一次远程更新的结果。
`update-agent-manifests` 立即拉取远程清单更新，重载到运行中的服务器，并打印更新后的清单状态；加 `--json` 获取原始状态响应。
`reload-agent-manifests` 在编辑本地覆盖后，把 Agent 检测清单重新加载到运行中的服务器。

## 通知

```bash
herdr notification show <title> [--body TEXT] [--position top-left|top-right|bottom-left|bottom-right] [--sound none|done|request]
```

`notification show` 使用配置的 `[ui.toast]` 投递方式。`--position` 只影响 Herdr 应用内的 toast。`--sound` 默认为 `none`；`done` 和 `request` 仅在通知实际显示时，播放已有的完成提示音和待处理提示音。

## Session（会话）

```bash
herdr session list [--json]
herdr session attach <name>
herdr session stop <name> [--json]
herdr session delete <name> [--json]
```

需要显式停止默认 Session 时，把名称写成 `default`。

## Workspace（工作区）

```bash
herdr workspace list
herdr workspace create [--cwd PATH] [--label TEXT] [--env KEY=VALUE] [--focus] [--no-focus]
herdr workspace get <workspace_id>
herdr workspace focus <workspace_id>
herdr workspace rename <workspace_id> <label>
herdr workspace close <workspace_id>
```

不抢占焦点创建 Workspace：

```bash
herdr workspace create --cwd ~/project --label api --no-focus
```

## Worktree

```bash
herdr worktree list [--workspace ID | --cwd PATH] [--json]
herdr worktree create [--workspace ID | --cwd PATH] [--branch NAME] [--base REF] [--path PATH] [--label TEXT] [--focus] [--no-focus] [--json]
herdr worktree open [--workspace ID | --cwd PATH] (--path PATH | --branch NAME) [--label TEXT] [--focus] [--no-focus] [--json]
herdr worktree remove --workspace ID [--force] [--json]
```

Worktree 是带有 Git 检出来源的普通 Herdr Workspace。`worktree create` 创建 Git worktree 检出，作为 Workspace 打开，并与父仓库 Workspace 归为一组。`--branch` 指定已有本地分支时，Herdr 检出该分支；否则从 `--base` 或 `HEAD` 创建新分支。不指定 `--path` 时，检出位置在 `<worktrees.directory>/<repo>/<branch-slug>` 下。

`workspace close` 只关闭 Herdr 侧的状态。`worktree remove` 是显式删除检出的命令：它执行 `git worktree remove`，从不删除分支，Git 因检出有未提交更改而拒绝时需要加 `--force`。

## Pane（窗格）

```bash
herdr pane list [--workspace <workspace_id>]
herdr pane current [--pane ID|--current]
herdr pane get <pane_id>
herdr pane layout [--pane ID|--current]
herdr pane process-info [--pane ID|--current]
herdr pane neighbor --direction left|right|up|down [--pane ID|--current]
herdr pane edges [--pane ID|--current]
herdr pane focus --direction left|right|up|down [--pane ID|--current]
herdr pane resize --direction left|right|up|down [--amount FLOAT] [--pane ID|--current]
herdr pane zoom [<pane_id>|--pane ID|--current] [--toggle|--on|--off]
herdr pane rename <pane_id> <label>|--clear
herdr pane split [<pane_id>|--pane ID|--current] --direction right|down [--ratio FLOAT] [--cwd PATH] [--env KEY=VALUE] [--focus] [--no-focus]
herdr pane swap --direction left|right|up|down [--pane ID|--current]
herdr pane swap --source-pane ID --target-pane ID
herdr pane move <pane_id> --tab <tab_id> --split right|down [--target-pane ID] [--ratio FLOAT] [--focus|--no-focus]
herdr pane move <pane_id> --new-tab [--workspace ID] [--label TEXT] [--focus|--no-focus]
herdr pane move <pane_id> --new-workspace [--label TEXT] [--tab-label TEXT] [--focus|--no-focus]
herdr pane close <pane_id>
```

对于接受 `--current` 的 Pane 命令，当命令在 Herdr Pane 内执行时，Herdr 使用调用方 Pane 的 `HERDR_PANE_ID`。对于 `pane split`，显式指定 Pane ID 或 `--pane ID` 会分割该 Pane，`--current` 分割调用方 Pane，省略目标时则使用当前 UI 焦点所在的 Pane。

读取输出：

```bash
herdr pane read <pane_id> [--source visible|recent|recent-unwrapped|detection] [--lines N]
herdr pane read <pane_id> --source visible --ansi
herdr pane read <pane_id> --source recent-unwrapped --lines 120
```

发送输入：

```bash
herdr pane send-text <pane_id> <text>
herdr pane send-keys <pane_id> <key> [key ...]
herdr pane run <pane_id> <command>
```

`<key>` 使用 Herdr 组合键语法：`a` 等可打印键，`enter`、`tab`、`esc`、`backspace`、`left`、`right`、`up`、`down` 等特殊键，`ctrl+h`、`control+j`、`alt+x`、`shift+tab` 等修饰组合键，`f1` 等功能键，以及 `minus`、`plus`、`backtick` 等命名标点。旧写法 `C-c` 和 `c-c` 也被接受，作为 `ctrl+c` 的别名。

`pane run` 把文本和回车作为原子操作提交。发送命令时优先用它，不要用 `send-text` 加 `send-keys Enter`。

从自定义钩子上报 Agent 状态：

```bash
herdr pane report-agent <pane_id> \
  --source ID \
  --agent LABEL \
  --state idle|working|blocked|unknown \
  [--message TEXT] \
  [--custom-status TEXT] \
  [--seq N] \
  [--agent-session-id ID] \
  [--agent-session-path PATH]
```

官方集成上报了原生 Session 引用时，`pane get`、`pane list`、`agent get` 和 `agent list` 会包含一个只读的 `agent_session` 对象。没有存储原生 Session 引用时，该字段省略。

Herdr 能解析控制 Pane 的前台进程 cwd 时，这些命令会包含 `foreground_cwd`。已有的 `cwd` 字段仍然是 Pane/Workspace 的 cwd，用于标签显示和 follow-cwd 行为。

`pane get` 和 `pane list` 在终端滚动指标可用时会包含 `scroll`。`scroll.offset_from_bottom == 0` 表示 Pane 处于回滚缓冲区的底部。

上报仅展示用的 Pane 元数据，不接管语义状态：

```bash
herdr pane report-metadata <pane_id> \
  --source ID \
  [--agent LABEL] \
  [--applies-to-source ID] \
  [--title TEXT|--clear-title] \
  [--display-agent TEXT|--clear-display-agent] \
  [--custom-status TEXT|--clear-custom-status] \
  [--state-label STATUS=TEXT] \
  [--clear-state-labels] \
  [--seq N] \
  [--ttl-ms N]
```

`STATUS` 取值为 `idle`、`working`、`blocked`、`done` 或 `unknown`。`--agent` 守卫权威 Agent 标签。`--applies-to-source` 守卫活动生命周期权威来源。用 `--display-agent` 修改显示名称。

元数据文本在存储前会规范化。Herdr 去掉首尾空白、移除控制字符，`--custom-status` 截断到 32 字符，`--title`、`--display-agent` 和每个 `--state-label` 的值截断到 80 字符。规范化后为空的值忽略。

`--source` 和 `--applies-to-source` 不超过 80 字符，且只能包含 ASCII 字母、数字、冒号、点、下划线和连字符。`--ttl-ms` 让元数据自动过期，取值范围 `1` 到 `86400000` 毫秒。想让元数据保留到被替换、清除或 Pane 关闭，就不传这个参数。`--seq` 让 Herdr 忽略来自同一 `--source` 的过期上报；过期上报会被 API 接受，但被 Pane 状态忽略。

## Agent（智能体）

```bash
herdr agent list
herdr agent get <target>
herdr agent read <target> [--source visible|recent|recent-unwrapped|detection] [--lines N] [--format text|ansi] [--ansi]
herdr agent send <target> <text>
herdr agent rename <target> <name>|--clear
herdr agent focus <target>
herdr agent wait <target> --status <idle|working|blocked|unknown> [--timeout MS]
herdr agent attach <target> [--takeover]
herdr agent start <name> [--cwd PATH] [--workspace ID] [--tab ID] [--split right|down] [--env KEY=VALUE] [--focus|--no-focus] -- <argv...>
herdr agent explain <target> [--json|--verbose]
herdr agent explain --file PATH --agent LABEL [--json|--verbose]
```

Agent 目标可以是终端 ID、唯一 Agent 名称、检测到或上报的 Agent 标签，以及旧式 Pane ID。名称和标签是 Agent 的身份标识。终端 ID 和旧式 Pane ID 是底层逃生舱。

`agent read` 读取解析后的终端流。`agent send` 向该流写入原始文本。`agent get`、`agent focus`、`agent wait` 和 `agent attach` 要求解析出的终端具备 Agent 身份。`agent rename` 可以赋予这个身份。

`agent explain` 请求运行中的服务器对屏幕检测所用的同一份底部缓冲区快照进行分类，因此输出实时反映服务器当前的清单缓存。它使用 `agent.explain` socket 方法，升级 Herdr 后，请先重启或交接到更新后的服务器，再使用实时 explain。

用 `--file PATH --agent LABEL` 可以在本地解释已保存的样本。

默认输出显示：Agent、最终状态、清单来源和版本、匹配的规则及其区域证据，以及回退、跳过或警告原因。加 `--verbose` 可以看到可见证据标志、缓存的远程版本、本地覆盖的遮蔽情况、远程更新状态，以及带匹配器和区域证据的完整评估规则列表。提交问题报告或写测试时加 `--json`。

普通终端、服务器、测试、shell 或底层终端控制，请使用 `pane send-text`、`pane send-keys`、`pane run` 和 `terminal attach`。需要带回车提交命令时用 `pane run`。

## 直接终端附加

```bash
herdr terminal attach <terminal_id> [--takeover]
herdr terminal session control <target> [--takeover] [--cols N] [--rows N]
herdr terminal session observe <target> [--cols N] [--rows N]
herdr terminal title set <title>
herdr terminal title clear
```

用 `ctrl+b q` 从直接附加中分离。用 `ctrl+b ctrl+b` 发送字面值 `ctrl+b`。

`terminal session control` 为 Pane、终端或 Agent 目标打开可写的实时终端流。它输出与 observe 模式相同的换行分隔 `terminal.frame` 和 `terminal.closed` 记录，从 stdin 读取换行分隔的 JSON 命令：`terminal.input`、`terminal.resize`、`terminal.scroll` 和 `terminal.release`。同一时刻一个终端只能有一个控制器；用 `--takeover` 替换现有控制器。

`terminal session observe` 为 Pane、终端或 Agent 目标打开只读的实时终端流。它输出换行分隔的 JSON `terminal.frame` 记录（含 base64 编码的 ANSI 字节），服务器关闭流时输出 `terminal.closed` 记录。多个观察者可以同时观看同一个终端，不抢占输入、调整大小、滚动或接管权限。

`terminal title clear` 恢复 Herdr 默认的外层终端窗口标题。

## 等待

等待 Pane 输出：

```bash
herdr wait output <pane_id> --match <text> [--source visible|recent|recent-unwrapped] [--lines N] [--timeout MS] [--regex] [--raw]
```

等待 Pane 的 Agent 状态：

```bash
herdr wait agent-status <pane_id> --status <idle|working|blocked|done|unknown> [--timeout MS]
```

普通命令和服务器用 `wait output`。AI 编程 Agent 用 `wait agent-status`。

## 插件

插件命令用于安装和运行本地可执行工作流插件。插件由清单加进程外命令组成；Herdr 负责宿主侧，插件自行选择实现语言。

安装、列出和移除插件：

```bash
herdr plugin install <owner>/<repo>[/subdir...] [--ref REF] [--yes]
herdr plugin list [--plugin ID] [--json]
herdr plugin uninstall <plugin_id|owner/repo[/subdir...]>
herdr plugin enable <plugin_id>
herdr plugin disable <plugin_id>
```

`plugin install` 只接受 GitHub 简写形式，如 `ogulcancelik/herdr-plugin-examples/worktree-bootstrap`。它调用 `git`，在交互式终端显示信任预览，运行受支持的清单构建命令，并将 GitHub 安装保存在 Herdr 管理的目录中。非交互式安装加 `--yes`。

重新安装 GitHub 管理的插件会替换该托管检出。不允许覆盖本地链接的插件。插件清单必须声明 `min_herdr_version`；插件要求更新版本的 Herdr 时，install 和 link 会失败。

`plugin list` 默认输出人类可读格式；要原始 API 响应传 `--json`。

本地开发：

```bash
herdr plugin link <path> [--disabled]
herdr plugin unlink <plugin_id>
```

`plugin link` 接受包含 `herdr-plugin.toml` 的插件目录，或直接的清单路径。从本地检出开发或测试插件时，用这个命令即可。`plugin unlink` 注销插件，不动文件。`plugin uninstall` 注销插件，同时删除 Herdr 管理的 GitHub 检出文件。

对于 GitHub 安装，uninstall 既接受插件 ID，也接受与 install 相同的 `owner/repo[/subdir...]` 简写。

动作、事件钩子、Pane 和链接处理器在清单中声明；运行时动作注册不在 v1 范围内。

配置目录：

```bash
herdr plugin config-dir <plugin_id>
```

`plugin config-dir` 打印插件的配置目录，需要时自动创建（存在旧版插件配置位置时会从中初始化）。在安装文档和 shell 脚本中用它给用户指出稳定路径，用于存放 `.env` 等用户可编辑配置，与托管的插件检出分开。

动作：

```bash
herdr plugin action list [--plugin ID]
herdr plugin action invoke <action_id> [--plugin ID]
```

`plugin action invoke` 为已安装、已启用、平台兼容的插件动作启动清单命令，并在 JSON 响应中打印该命令的日志记录。多个插件使用相同动作 ID 时，使用限定动作 ID（`plugin.id.action`）。本地动作 ID 不能包含点，因此即使插件 ID 含点，限定 ID 也不会有歧义。

日志：

```bash
herdr plugin log list [--plugin ID] [--limit N]
```

托管终端 Pane：

```bash
herdr plugin pane open --plugin ID --entrypoint ID [--placement overlay|split|tab|zoomed] [--workspace ID] [--target-pane PANE] [--direction right|down] [--cwd PATH] [--env KEY=VALUE] [--focus|--no-focus]
herdr plugin pane focus <pane_id>
herdr plugin pane close <pane_id>
```

`plugin pane open` 要求插件已链接、已启用并兼容当前平台。它将清单中声明的 `[[panes]]` 命令作为 Herdr 管理的终端 Pane 启动。清单默认值是 `overlay`，在活动 Pane 上方打开一个临时缩放覆盖层。也可以作为分割、新 Tab 或缩放 Pane 打开。非终端的原生插件 Pane 是后续能力。

`--env KEY=VALUE` 可在启动进程的命令上重复使用，仅作用于新启动的进程。与调用方提供的环境变量冲突时，`HERDR_SOCKET_PATH`、`HERDR_BIN_PATH`、`HERDR_ENV`、`HERDR_WORKSPACE_ID`、`HERDR_TAB_ID`、`HERDR_PANE_ID`、`HERDR_PLUGIN_ID`、`HERDR_PLUGIN_ROOT`、`HERDR_PLUGIN_CONFIG_DIR`、`HERDR_PLUGIN_STATE_DIR`、`HERDR_PLUGIN_ENTRYPOINT_ID` 和 `HERDR_PLUGIN_CONTEXT_JSON` 等 Herdr 管理的变量保持权威。

## 读取来源

| 来源 | 含义 |
| --- | --- |
| `visible` | 当前渲染的屏幕。最适合 UI 反馈循环。 |
| `recent` | 带终端折行的最近回滚内容。 |
| `recent-unwrapped` | 不带软折行的最近回滚内容。最适合日志。 |
| `detection` | Agent 屏幕检测使用的底部缓冲区快照。 |

## 环境变量

| 变量 | 用途 |
| --- | --- |
| `HERDR_CONFIG_PATH` | 覆盖配置文件路径。 |
| `HERDR_SESSION` | 为 CLI 命令选择命名 Session。 |
| `HERDR_SOCKET_PATH` | 底层 socket 路径覆盖。 |
| `HERDR_ENV` | 在 Herdr 管理的 Pane 进程内设为 `1`。 |
| `HERDR_PANE_ID` | 运行中 Pane 进程的公开 Pane ID。 |
| `HERDR_TAB_ID` | 运行中 Pane 进程的公开 Tab ID。 |
| `HERDR_WORKSPACE_ID` | 运行中 Pane 进程的公开 Workspace ID。 |
| `HERDR_LOG` | 设置日志过滤，例如 `HERDR_LOG=herdr=debug`。 |
| `HERDR_DISABLE_SOUND` | 即使启用声音通知也禁用声音播放。 |

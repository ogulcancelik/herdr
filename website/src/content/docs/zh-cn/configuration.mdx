Herdr 无需配置文件即可使用。想自定义按键、主题、侧边栏设置、通知或高级行为时，再加一个就行。

## 配置文件

Herdr 从以下路径读取配置：

```text
Linux 和 macOS: ~/.config/herdr/config.toml
Windows:         %APPDATA%\herdr\config.toml
```

运行 `herdr --help` 可查看系统解析后的配置文件路径。

打印完整的默认配置：

```bash
herdr --default-config
```

想要一个完整的起点，可以把它保存为你的配置：

```bash
herdr --default-config > ~/.config/herdr/config.toml
```

如果某个配置值无效，Herdr 会回退到安全默认值，并在启动时显示警告。

当 `onboarding` 缺失或为 true 时，Herdr 会显示首次运行引导。完成引导后会写入 `onboarding = false` 并打开设置中的集成 Tab。安装完成后想跳过该流程，可以手动设置。

```toml
onboarding = false
```

## 更新

Linux 和 macOS 的直接安装默认使用稳定更新通道。Windows 测试版安装默认使用预览通道，在稳定版 Windows 发布前无法切换到稳定通道。

```toml
[update]
channel = "stable"
version_check = true
manifest_check = true
```

设置 `channel = "preview"` 可以让 `herdr update` 安装从当前开发分支手动发布的预览版本。Homebrew、mise 和 Nix 安装会忽略预览通道，通过各自的包管理器更新。

设置 `version_check = false` 可禁用后台检查 Herdr 新版本。手动执行 `herdr update` 仍会使用配置的通道。

设置 `manifest_check = false` 可禁用后台远程 Agent 检测清单检查。Herdr 仍会使用内置清单和本地覆盖。

## 重载配置

编辑 `config.toml` 后，重载运行中的服务器：

```bash
herdr server reload-config
```

也可以在 Herdr 中打开全局菜单，选择 `reload config`。

重载会在不重启 Pane 的情况下应用大多数 UI 设置。仅在启动时生效的设置仍需重启。

## 终端默认值

设置 Herdr 为新建交互式 Pane 使用的可执行文件：

```toml
[terminal]
default_shell = "nu"
```

未设置或为空时，Herdr 使用 `$SHELL`，Unix 系统上再回退到 `/bin/sh`，Windows 上回退到 PowerShell。这里填的是可执行文件名或路径，不是 shell 命令行。现有 Pane 在重建前保持当前 shell。

自定义命令按键绑定字符串在 Unix 上通过 `/bin/sh -c` 运行 Pane 命令，通过 `/bin/sh -lc` 运行分离式命令；Windows 上通过 `cmd.exe /d /c` 运行。

设置 Herdr 如何启动新建交互式 Pane 的 shell：

```toml
[terminal]
shell_mode = "auto"
```

`shell_mode = "auto"` 在 macOS 上启动登录 shell，让 `/usr/libexec/path_helper` 这类仅登录时生效的 PATH 设置和 Homebrew shell 初始化能在新 Pane 中运行。其他平台保持现有的非登录 shell 行为。用 `"login"` 强制登录 shell 启动，用 `"non_login"` 关闭。命令 Pane、分离式自定义命令按键绑定和显式 argv 启动保持原有的命令执行路径。

设置新 Pane、Tab 和 Workspace 的工作目录策略：

```toml
[terminal]
new_cwd = "follow"
```

`new_cwd = "follow"` 保持默认行为，继承来源 Pane 或 Workspace 的目录。没有来源 Workspace 时，Herdr 从 `$HOME` 启动。用 `"home"` 总是从 `$HOME` 启动，用 `"current"` 使用 Herdr 进程目录，或用 `"~/Projects"` 这类固定路径。CLI 或 Socket API 传入的显式 `--cwd` 值仍然优先。

## Worktree

设置从侧边栏创建 Git worktree 检出时，Herdr 使用的根目录：

```toml
[worktrees]
directory = "~/.herdr/worktrees"
```

Herdr 在 `<directory>/<repo>/<branch-slug>` 下创建检出。想要同级目录风格的检出，把它设为 `~/Projects/herdr-worktrees` 这类目录。应用配置时，相对路径会解析为绝对路径。

Worktree 操作在 Git Workspace 行上可用。`New worktree` 会创建一个检出：如果输入的分支名已存在，检出该本地分支；否则创建新分支。然后作为新的 Herdr Workspace 打开，并归到来源 Workspace 之下。`Open worktree...` 列出该仓库已有的 Git worktree 检出。选择已打开的检出会聚焦到它，选择已关闭的检出会在同一组中打开它。

归组的 worktree 行为和普通 Herdr Workspace 一样：可以聚焦、重命名、关闭，拥有自己的 Tab 和 Pane。父行是原始 Workspace。关闭父行会关闭整个 Herdr 组，但不会删除检出文件夹或分支。

删除 worktree 检出需要显式操作。在归组的子 Workspace 上使用 `Delete worktree checkout...` 来执行 `git worktree remove`。Herdr 先请求 Git 安全删除。如果 Git 因为检出有修改或未跟踪文件而拒绝，Herdr 会在执行强制删除前再次确认。分支不会被删除。

## 远程连接

远程连接默认使用带临时保活兜底的方式管理 SSH 连接。

```toml
[remote]
manage_ssh_config = true
```

启用后，`herdr --remote` 会写入一个私有的临时 SSH 配置：先包含你的 `~/.ssh/config` 和 `/etc/ssh/ssh_config`，再添加兜底的 `ServerAliveInterval` 和 `ServerAliveCountMax` 值。你自己的保活设置优先。Herdr 还会为每次连接使用一个私有的 OpenSSH 控制套接字，复用首次认证的连接。设置 `manage_ssh_config = false` 则通过普通 `ssh` 运行远程连接，不使用 Herdr 生成的配置或控制套接字。

## 按键绑定

前缀键的入门介绍和经过验证的免前缀配置，参见[键盘](/zh-cn/docs/keyboard/)。

Herdr 有类似 tmux 的前缀模式。默认前缀是 `ctrl+b`。按键绑定字符串是显式的：`prefix+n` 表示先按配置的前缀再按 `n`；`ctrl+alt+n` 是终端模式下的直接快捷键。

一个小型按键绑定覆盖示例：

```toml
[keys]
prefix = "ctrl+b"
goto = "prefix+g"
new_tab = "prefix+c"
next_tab = "prefix+n"
previous_tab = "prefix+p"
focus_pane_left = "prefix+h"
navigate_workspace_down = "j"
navigate_pane_down = "ctrl+j"
split_horizontal = "prefix+minus"
```

默认键位以前缀为先，避免直接快捷键抢占 shell、编辑器、tmux 或终端应用的输入。常见默认值包括：

```toml
[keys]
detach = "prefix+q"
workspace_picker = "prefix+w"
goto = "prefix+g"
new_workspace = "prefix+shift+n"
new_worktree = "prefix+shift+g"
rename_workspace = "prefix+shift+w"
close_workspace = "prefix+shift+d"
navigate_workspace_up = "up"
navigate_workspace_down = "down"
navigate_pane_left = "h"
navigate_pane_down = "j"
navigate_pane_up = "k"
navigate_pane_right = "l"
remote_image_paste = "ctrl+v"
new_tab = "prefix+c"
previous_tab = "prefix+p"
next_tab = "prefix+n"
switch_tab = "prefix+1..9"
rename_tab = "prefix+shift+t"
close_tab = "prefix+shift+x"
copy_mode = "prefix+["
focus_pane_left = "prefix+h"
focus_pane_down = "prefix+j"
focus_pane_up = "prefix+k"
focus_pane_right = "prefix+l"
swap_pane_left = "prefix+shift+h"
swap_pane_down = "prefix+shift+j"
swap_pane_up = "prefix+shift+k"
swap_pane_right = "prefix+shift+l"
cycle_pane_next = "prefix+tab"
cycle_pane_previous = "prefix+shift+tab"
last_pane = ""
split_vertical = "prefix+v"
split_horizontal = "prefix+minus"
close_pane = "prefix+x"
zoom = "prefix+z"
resize_mode = "prefix+r"
toggle_sidebar = "prefix+b"
```

可选动作默认未设置。用 `prefix+` 绑定可获得前缀模式行为；如果确实需要直接快捷键，可以用显式的修饰组合键。

```toml
[keys]
previous_workspace = "prefix+shift+left"
next_workspace = "prefix+shift+right"
last_pane = "prefix+tab"
open_worktree = "prefix+shift+o"
remove_worktree = "prefix+alt+d"
next_tab = ["prefix+n", "ctrl+alt+]"]
```

`last_pane` 跨 Workspace 和 Tab 切回上一个聚焦的 Pane。它默认未设置，因为 tmux 风格的 Pane 绑定 `prefix+l` 已被用于向右聚焦 Pane。

`remote_image_paste` 只在 `herdr --remote` 中生效。它是本地客户端的快捷键，把本地剪贴板图像发送到远程 Pane。设为空字符串可禁用这个原始按键快捷键；外层终端发送的 paste-image 信号仍然有效。

按键字符串接受普通按键、`ctrl+a`、`shift+n`、`alt+1`、`cmd+k` 这类修饰组合，以及 `enter`、`tab`、`esc`、`left`、`right`、`up`、`down` 这类特殊键。也接受 `minus`、`comma`、`ampersand`、`plus`、`backtick` 这类命名标点。

像 `n` 这样的普通可打印键直接绑定不安全——它会拦截打字输入。除非有意要直接绑定，否则用 `prefix+n`。

`navigate_workspace_*` 和 `navigate_pane_*` 字段仅在导航模式下生效，可以用 `j`、`k` 这类普通键；但不能使用 `prefix+`、`esc`、`enter`、`tab`、`shift+tab`、`left`、`right` 或无修饰的 `1` 到 `9`。左右方向键是 Pane 左/右导航的永久别名。

这些导航模式快捷键与 `focus_pane_down = "prefix+j"` 这类通用动作绑定相互独立。两者使用同一个键时，导航模式打开期间导航模式的快捷键优先。

Alt、Cmd/Super 和带修饰键的标点是否可用，取决于你的终端和 tmux 设置。

如果你有旧的自定义按键绑定，想使用新的默认值，运行 `herdr config reset-keys`。Herdr 会备份 `config.toml`，移除 `[keys]` 和 `[[keys.command]]`，重启或执行 `herdr server reload-config` 后使用内置的 v2 默认值。

## 索引跳转

索引式按键绑定在普通按键绑定字段中使用 `1..9`：

```toml
[keys]
switch_tab = "prefix+1..9"
switch_workspace = "prefix+shift+1..9"
focus_agent = "prefix+alt+1..9"
```

旧式的 `[keys.indexed]` 表出于兼容仍会被解析，但新配置应优先使用显式的动作字段。

## 自定义命令按键绑定

自定义命令使用同样的按键绑定语法。

```toml
[[keys.command]]
key = "prefix+alt+g"
type = "pane"
command = "lazygit"
description = "run lazygit"
```

`type = "pane"` 打开一个临时 Pane，命令退出时自动关闭。

`type = "shell"` 在后台分离运行。

`type = "plugin_action"` 调用已安装插件的动作 id。动作 id 不全局唯一时使用限定 id：

```toml
[[keys.command]]
key = "prefix+l"
type = "plugin_action"
command = "example.layout.apply"
description = "apply layout"
```

可以提供可选的 `description`。指定后，它会显示在按键帮助面板（按 `prefix+?` 打开）中，替代默认的 `'custom command'` 标签。

自定义命令在可用时会收到 `HERDR_SOCKET_PATH`、`HERDR_BIN_PATH`、`HERDR_ACTIVE_WORKSPACE_ID`、`HERDR_ACTIVE_TAB_ID`、`HERDR_ACTIVE_PANE_ID` 和 `HERDR_ACTIVE_PANE_CWD`。Herdr 能检测到时，shell 命令从聚焦 Pane 的工作目录运行。

Windows 上的自定义命令字符串通过 `cmd.exe /d /c` 执行，所以环境变量要用 `%HERDR_BIN_PATH%` 语法。要运行 PowerShell 语法，需显式调用，例如 `powershell.exe -NoProfile -Command "..."`。

## 主题

选择内置主题：

```toml
[theme]
name = "catppuccin"
```

内置主题：

`catppuccin`、`catppuccin-latte`、`terminal`、`tokyo-night`、`tokyo-night-day`、`dracula`、`nord`、`gruvbox`、`gruvbox-light`、`one-dark`、`one-light`、`solarized`、`solarized-light`、`kanagawa`、`kanagawa-lotus`、`rose-pine`、`rose-pine-dawn`、`vesper`。

想让 Herdr UI 颜色跟随宿主终端的 ANSI 调色板时，用 `terminal`。

要让 Herdr 在宿主终端报告明暗外观变化时自动切换 UI 主题，启用主题自动切换：

```toml
[theme]
name = "catppuccin"
auto_switch = true
light_name = "catppuccin-latte"
dark_name = "catppuccin"
```

`auto_switch` 默认为 `false`，所以现有主题配置保持手动行为。省略 `light_name` 或 `dark_name` 时，如果配置的 `name` 有对应的内置姊妹主题，Herdr 会自动使用——比如 `tokyo-night`/`tokyo-night-day` 或 `gruvbox`/`gruvbox-light`。在设置中手动选择主题会禁用 `auto_switch`。

你可以覆盖单个颜色：

```toml
[theme.custom]
panel_bg = "reset"
accent = "#a6e3a1"
green = "#a6e3a1"
blue = "#89b4fa"
red = "#f38ba8"
yellow = "#f9e2af"
```

颜色值接受十六进制、命名颜色、`rgb(r,g,b)`，或 `reset`、`default`、`none`、`transparent` 这类重置别名。

## UI 与侧边栏

侧边栏是 Herdr 的主仪表盘。它显示 Workspace、Tab、Pane 和 Agent 状态。

常用选项：

```toml
[ui]
sidebar_width = 32
sidebar_min_width = 18
sidebar_max_width = 36
sidebar_collapsed_mode = "compact"
mobile_width_threshold = 64
mouse_capture = true
host_cursor = "auto"
right_click_passthrough_modifier = ""
redraw_on_focus_gained = true
mouse_scroll_lines = 3
confirm_close = true
prompt_new_tab_name = true
pane_borders = true
pane_gaps = true
show_agent_labels_on_pane_borders = false
hide_tab_bar_when_single_tab = false
agent_panel_sort = "spaces"
accent = "cyan"
```

`sidebar_min_width` 和 `sidebar_max_width` 以列数控制展开侧边栏的调整范围。默认是 18 和 36。

`sidebar_collapsed_mode` 控制侧边栏关闭后保留什么内容。默认值 `compact` 保留窄状态条。设为 `hidden` 可让收起的侧边栏宽度为零；通过 `keys.toggle_sidebar` 重新打开。

`mobile_width_threshold` 控制 Herdr 使用移动端单列布局的终端宽度阈值。默认 64 列；折叠屏、平板或宽屏手机终端可以调大。

Agent 面板显示所有空间中的所有 Agent。`agent_panel_sort` 可以是 `spaces` 或 `priority`；`workspaces` 作为 `spaces` 的别名也被接受。默认是 `spaces`，让 Agent 按空间顺序分组。`priority` 按关注优先级排序：blocked、done、working、idle、unknown。同一状态内，最近改变状态的 Agent 排在前面。

`confirm_close` 控制关闭 Workspace 时是否需要确认。`prompt_new_tab_name` 控制新建 Tab 时是否先询问名称。

如果想让终端处理普通点击（比如 command+点击 URL），设置 `mouse_capture = false`。鼠标捕获启用时，如果终端把带修饰键的点击传给 Herdr，Ctrl+点击可以打开 Pane 链接。macOS 上同理；Herdr 捕获鼠标输入期间，Cmd+点击不会与普通点击区分上报。终端原生的绕过路径：Linux 上用 Shift+Ctrl+点击，macOS 上用 Shift+Cmd+点击。

`host_cursor` 控制 Herdr 使用外层终端的原生光标，还是把光标绘制为终端单元格内容。默认值 `auto` 在 Windows 上绘制 Herdr 光标，以避免 ConPTY 光标在活动重绘期间闪烁；其他平台使用原生光标。设为 `native` 始终使用外层终端光标，设为 `drawn` 始终使用 Herdr 绘制的光标。

如果想让 Ctrl+右键点击、按住和拖动手势到达启用鼠标上报的 Pane 应用，而不是打开 Herdr 的 Pane 菜单，设置 `right_click_passthrough_modifier = "ctrl"`。默认为空，即禁用这种透传。支持的修饰键有 `ctrl`、`alt`、`cmd`、`super`、`meta` 和 `hyper`；`shift` 被拒绝，因为许多终端把 Shift+鼠标保留给它们自己的鼠标绕过。

设置 `redraw_on_focus_gained = false` 可避免切回 Herdr 时可见的全屏刷新。默认为 `true`，因为完整重绘能从罕见的过期或脏宿主终端画面中恢复。

设置 `mouse_scroll_lines` 改变鼠标滚轮每格滚动的 Pane 回滚行数。默认是 3。请求鼠标上报的 Pane 应用仍直接接收滚轮事件。备用屏幕应用没有可滚动的回滚内容，参见[回滚缓冲区](#回滚缓冲区)。

设置 `pane_borders = false` 移除分割 Pane 边框。设置 `pane_gaps = false` 让分割 Pane 共用紧凑的分隔线。禁用边框时，Pane 间距为一个空白终端单元格。终端单元格通常高大于宽，所以上下 Pane 之间一行的间距可能看起来比左右 Pane 之间一列的间距更宽。

如果想在没有手动 Pane 标签时，把检测到的 Agent 标签显示在分割 Pane 边框上，设置 `show_agent_labels_on_pane_borders = true`。

设置 `hide_tab_bar_when_single_tab = true` 后，当活动 Workspace 只有一个 Tab 时隐藏 Tab 栏。仍可通过配置的按键绑定创建新 Tab。出现第二个 Tab 时，Herdr 恢复 Tab 栏并调整 Pane 大小腾出空间。

## 通知

Herdr 可以在 Agent 完成或需要输入时显示弹出通知。

```toml
[ui.toast]
delivery = "off"
delay_seconds = 1

[ui.toast.herdr]
position = "bottom-right"

[ui.toast.clipboard]
enabled = true
position = "bottom-center"
```

`delivery = "off"` 禁用弹出通知。这是默认值。

`delivery = "herdr"` 在 Herdr UI 内显示 toast。点击 toast 或绑定 `keys.open_notification_target`，可以聚焦目标 Workspace、Tab 和 Pane。`ui.toast.herdr.position` 可设为 `top-left`、`top-right`、`bottom-left` 或 `bottom-right`；位置相对于整个 Herdr 窗口。

`delivery = "terminal"` 请求外层终端显示桌面通知。Herdr 为 Ghostty、iTerm2、Kitty 和 WezTerm 发送终端通知转义序列。通过 SSH 连接时很有用，因为通知归本地终端所有。

`delivery = "system"` 直接请求本地操作系统。macOS 上，Herdr 优先使用 `terminal-notifier`，不可用时回退到 `/usr/bin/osascript`。点击 `terminal-notifier` 的通知可以激活承载的终端。Linux 上，Herdr 使用 `notify-send`，需要 `DISPLAY` 或 `WAYLAND_DISPLAY`。

弹出通知用于后台提醒。Herdr 对活动 Tab 抑制弹出。

`delay_seconds` 在发送完成或需要输入的 Agent 通知前等待。只有延迟结束时 Pane 仍处于相同状态，Herdr 才会通知。设为 `0` 立即通知。有效值为 `0` 到 `3600`。

剪贴板反馈单独配置，因为它确认的是前台复制操作，且从不通过 terminal 或 system 投递。设置 `ui.toast.clipboard.enabled = false` 隐藏"已复制到剪贴板"弹窗。剪贴板位置有 `top-left`、`top-center`、`top-right`、`bottom-left`、`bottom-center` 和 `bottom-right`。

## 声音

声音通知默认启用，由本地 Herdr 客户端播放。

```toml
[ui.sound]
enabled = true
```

Herdr 在 Agent 完成时播放完成音，在 Agent 需要输入时播放提醒音。在共享机器或远程服务器上，除非明确想要声音，否则设置 `enabled = false`。

macOS 上，Herdr 使用 `afplay`。Linux 上，Herdr 按顺序尝试支持 mp3 的播放器：`paplay`、`pw-play`、`ffplay`、`mpg123`、`mpv`。没有可用播放器时，声音播放被跳过，Herdr 记录一条警告。

自定义声音必须是 mp3 文件。相对路径从配置文件所在目录解析。

```toml
[ui.sound]
path = "sounds/notification.mp3"
done_path = "sounds/done.mp3"
request_path = "sounds/request.mp3"
```

`path` 为所有声音通知设置一个声音。`done_path` 和 `request_path` 只覆盖完成音和需要输入音。

按 Agent 的声音覆盖接受 `default`、`on` 或 `off`。用检测到的 Agent 标签做键，比如 `claude`、`codex`、`devin` 或 `droid`。Droid 默认静音。

```toml
[ui.sound.agents]
droid = "off"
claude = "on"
```

## 回滚缓冲区

设置新建 Pane 的回滚缓冲区大小：

```toml
[advanced]
scrollback_limit_bytes = 10485760
```

现有 Pane 在重建前保持当前缓冲区。

只有应用写入主屏幕时，Pane 才显示滚动条。切换到备用屏幕的全屏应用（vim、htop、设置了 `CLAUDE_CODE_NO_FLICKER=1` 的 Claude Code）不产生回滚内容，不显示滚动条，滚轮事件直接路由给应用。请用应用自己的按键或 UI 滚动。

## Pane（窗格）屏幕历史

默认情况下，完整 Session 重启会恢复 Workspace、Tab、Pane、cwd、布局和焦点，但不保存 Pane 内容。

Pane 屏幕历史默认关闭。Pane 输出可能包含密钥、令牌、提示词和命令输出，所以只在你确实需要让 Herdr 跨完整服务器重启保存最近 Pane 内容时才开启：

```toml
[experimental]
pane_history = true
```

也可以在 Settings > Experiments > pane screen history 中切换。

开启后，Herdr 把保存的 Pane 历史存放在 `session.json` 旁边的 `session-history.json` 中。

Pane 屏幕历史与实时持久化、快照恢复、Agent 原生 Session 恢复和实时交接的区别，参见[Session 状态与恢复](/zh-cn/docs/session-state/)。

## Agent Session 恢复

Herdr 可以在服务器重启后，让受支持的 Agent Pane 在其原生对话 Session 中重新启动。

```toml
[session]
resume_agents_on_restore = true
```

这默认开启。Herdr 只恢复通过官方 Herdr 集成上报了原生 Session 引用的 Pane。受支持的恢复目标包括 Claude Code、Codex、Cursor Agent CLI、GitHub Copilot CLI、Droid、Kimi Code CLI、Qoder CLI、Pi、Hermes Agent、OpenCode、Kilo Code CLI 和 MastraCode。不受支持、缺失、无效、重复或过期的 Session 引用，会在保存的 Pane 目录中作为普通 shell 恢复。

Session 引用存储在本地 Herdr Session 快照中。它们不会出现在普通的 Pane、Agent、状态或事件输出中。

Agent 原生 Session 恢复与 Pane 屏幕历史和实时交接的区别，参见[Session 状态与恢复](/zh-cn/docs/session-state/)。

## IME 光标跟踪

当聚焦 Pane 隐藏光标并自己绘制光标时——这在 Claude Code、pi、codex、Devin 这类 AI 编程 Agent TUI 中很常见——macOS 原生输入法会停止跟踪候选窗口位置，因为外层终端不再报告光标。

设置 `reveal_hidden_cursor_for_cjk_ime = true`，无视 Pane 的 `?25l` 请求，把聚焦 Pane 的光标锚点暴露给外层终端：

```toml
[experimental]
reveal_hidden_cursor_for_cjk_ime = false
cjk_ime_agents = []
cjk_ime_cursor_shape = "steady_block"
```

启用后，光标保持在聚焦 Pane 上报的位置可见。如果 Pane 没有上报光标位置，锚点回退到 Pane 左上角，保证始终有稳定的输入法提示位置。

`cjk_ime_agents` 是可选的允许列表。为空时，该功能适用于任何聚焦 Pane。非空时，只在聚焦 Pane 检测到的 Agent 匹配列表中某个名称时才生效——适合只对自绘光标的 AI Agent TUI 启用，而不影响普通 shell。接受的名称：`pi`、`claude`、`codex`、`gemini`、`cursor`、`devin`、`agy`、`cline`、`opencode`、`copilot`、`kimi`、`kiro`、`droid`、`amp`、`grok`、`hermes`、`kilo`、`qodercli` 和 `qoder`。未知名称被忽略；如果列表中没有有效名称，该功能不生效。

`cjk_ime_cursor_shape` 控制为输入法锚点渲染的 DECSCUSR 形状。接受的值：`block`、`steady_block`（默认）、`underline`、`steady_underline`、`bar`、`steady_bar`。

通过已有的 `[experimental]` 块热重载。

启用的代价：对隐藏光标但不绘制替代光标的应用（vim 普通模式等），外层终端会多显示一个硬件光标。配合 `cjk_ime_agents` 可以把它限定到特定 TUI。

## 前缀输入法切换

在 macOS 上，当非 ASCII 输入法处于活动状态时，前缀模式命令可能很难使用，因为前缀命令仍会经过宿主输入法解释。

设置 `switch_ascii_input_source_in_prefix = true`，在前缀模式激活期间把宿主输入法切换到系统的 ASCII 输入法：

```toml
[experimental]
switch_ascii_input_source_in_prefix = false
```

启用后，Herdr 在进入前缀模式后切换输入法，在导航、菜单、调整大小、复制模式等前缀启动的模式中保持 ASCII 输入法，回到终端或进入重命名对话框等文本字段时恢复之前的输入法。该设置仅限 macOS，在其他平台或系统输入法切换失败时不做任何事。

也可以在 Settings > Experiments > switch to ascii input source in prefix (macOS) 中切换。

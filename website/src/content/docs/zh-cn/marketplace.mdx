title: 插件市场
description: 在 GitHub 上发现社区 Herdr 插件，并让你自己的插件被收录。
---

Herdr 插件市场收录社区插件，方便你按需发现。前往 [herdr.dev/plugins](/plugins/) 浏览。它自动索引公开的 GitHub 仓库，不经过人工审核。

## 浏览插件

[插件市场](/plugins/)列出所有打上 GitHub 主题 `herdr-plugin` 的公开仓库。你可以按名称、作者、描述或语言搜索，也可以按热度、近期活跃度或最新排序。每个条目直接链接到 GitHub 上的源码仓库。

收录是自动的，不经过审核。出现在列表里只说明仓库给自己打了标签，不代表 Herdr 审查过。安装任何插件前，请先阅读[信任指南](/zh-cn/docs/plugins/#信任与安全)。

## 安装插件

插件市场解决的是发现问题，安装方式不变。任何插件都可以直接从 GitHub 安装：

```bash
herdr plugin install owner/repo[/subdir...]
```

只要发布一个公开的 GitHub 仓库，在根目录或子目录放一份 `herdr-plugin.toml` 清单，这条命令就能用。清单格式和编写参考见[插件](/zh-cn/docs/plugins/)。

## 让你的插件被收录

给公开仓库加上 GitHub 主题 `herdr-plugin`。索引只认这一个信号，打上标签就行。索引每 30 分钟自动刷新一次，新打标的仓库很快出现，去掉标签的仓库会在下一次刷新时移除。

## 条目会展示什么

每张卡片显示 GitHub 仓库的元数据：仓库名和作者、描述、star 数、主要语言、最后 push 时间，以及指向源码的链接。索引从 GitHub 仓库搜索接口读取这些信息，所以保持仓库描述和主题准确，你的条目才好用。

索引目前还不解析 `herdr-plugin.toml`，所以插件 `id`、声明的 `platforms`、`min_herdr_version` 等清单字段在 v1 中不会展示。Fork 和已归档的仓库不会出现在列表里。
